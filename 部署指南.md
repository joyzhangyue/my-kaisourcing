# 网站部署指南

## 你需要准备的东西

- 一个域名（例如 `kaisource.com`）
- 一张信用卡/支付宝（买域名用）
- 一个 GitHub 账号（免费注册：github.com）

---

## 第一步：买域名（10 分钟）

推荐以下几家的任一个：

| 平台 | 适合场景 | 优点 |
|------|---------|------|
| **Namesilo** | 海外客户为主 | 便宜、隐私保护免费 |
| **GoDaddy** | 通用 | 中文界面 |
| **阿里云/腾讯云** | 国内客户为主 | 国内速度快，后续备案方便 |

**怎么买：**
1. 去任意一家搜你想要的域名（`kaisource.com`、`yoursourcing.cn` 等等）
2. 加入购物车，付款
3. 付款后在后台找到 **DNS 管理 / Nameserver 设置** 页面

**先别急着改 DNS，等第三步用到。**

---

## 第二步：把网站上传到 GitHub（15 分钟）

如果你没有 GitHub 账号，先去 [github.com](https://github.com) 注册一个。

1. 登录 GitHub，点右上角 **+** → **New repository**
2. Repository name 随便填，比如 `my-sourcing-site`，选 **Public**，点 **Create repository**
3. 进入你的网站文件夹，在地址栏输入 `cmd` 回车打开命令行，依次执行：

```bash
git init
git add .
git commit -m "first commit"
git branch -M main
git remote add origin https://github.com/你的用户名/my-sourcing-site.git
git push -u origin main
```

> 第一次运行会让你输入 GitHub 用户名和密码（密码用 Personal Access Token，在 GitHub Settings → Developer settings → Personal access tokens 里生成）

---

## 第三步：用 Cloudflare Pages 部署（15 分钟）

1. 去 [cloudflare.com](https://cloudflare.com) 注册账号
2. 登录后，左侧菜单点 **Pages**
3. 点 **Create a project** → **Connect to Git**
4. 授权 Cloudflare 访问你的 GitHub，然后选中刚才建的仓库 `my-sourcing-site`
5. 在配置页面：
   - **Project name**：随便填
   - **Production branch**：`main`
   - **Build settings**：选 **None**（因为我们没有 build 步骤）
   - **Build output directory**：留空
6. 点 **Save and Deploy**
7. 等一两分钟，部署完成，你会得到一个 `xxx.pages.dev` 的网址，点进去应该能看到你的网站了

---

## 第四步：绑定你的域名（10 分钟）

1. 在 Cloudflare Pages 的项目页面，点 **Custom domains** → **Set up a custom domain**
2. 输入你买的域名（比如 `kaisource.com`），点 **Continue**
3. 它会告诉你需要去你的域名注册商那里改 Nameserver

**回到你买域名的网站：**
1. 找到 DNS 管理 / Nameserver 设置
2. 把原来的 Nameserver 替换成 Cloudflare 给你的那两个（类似 `ns1.cloudflare.com` 和 `ns2.cloudflare.com`）
3. 保存

回到 Cloudflare，等几分钟到几小时，状态变成 **Active** 就说明生效了。

---

## 第五步：设置企业邮箱（可选，10 分钟）

用 Cloudflare 的免费邮件转发，让你拥有 `you@kaisource.com` 这样的邮箱：

1. 在 Cloudflare 后台，点你的域名
2. 左侧 **Email** → **Email Routing**
3. 开启 Email Routing
4. 点 **Add destination address**，输入你的私人邮箱（比如你的 Gmail）
5. 点 **Catch-All Addresses** → **Edit**，选择刚才添加的地址

以后发到 `你@你的域名.com` 的邮件，都会自动转发到你的私人邮箱。

---

## 以后怎么更新网站

1. 在本地修改 `index.html` 或 `css/style.css`
2. 打开命令行运行：

```bash
git add .
git commit -m "更新了什么内容"
git push
```

3. Cloudflare Pages 会自动检测到更新，重新部署。等一两分钟，网站就会自动更新。

---

## 如果客户主要在中国

用阿里云方案（需备案）：
1. 在阿里云买虚拟主机（共享版就行，¥200-300/年）
2. 在阿里云提交网站备案（需要几天时间）
3. 备案通过后，用 FTP 工具（比如 FileZilla）把网站文件上传到虚拟主机的 `htdocs` 目录
4. 在阿里云 DNS 解析里把域名指向虚拟主机的 IP

## 常见问题

**问：DNS 要等多久才能生效？**
答：一般几分钟到几小时，全球生效最长可能需要 24 小时。

**问：以后想加联系表单后台功能怎么办？**
答：可以加，到时候找我就行。

**问：网站内容怎么改？**
答：直接告诉我，我帮你改好，你再按 "以后怎么更新" 那步推上去就行。
